# HappyPC — интернет-магазин компьютерных комплектующих

Полноценный веб-сайт для подбора и покупки компьютерных комплектующих с функцией сборки ПК, корзиной, оформлением заказа, личным кабинетом, избранным и обратной связью.

## 🚀 Технологический стек

### Бэкенд

- **Go** (1.26)
- **Chi** — маршрутизатор
- **GORM** — ORM
- **PostgreSQL** — база данных
- **bcrypt** — хеширование паролей
- **JWT‑подобные сессии** (через cookie и таблицу sessions)

### Фронтенд

- **Vue 3** (Composition API)
- **Vite** — сборка
- **Vue Router** — маршрутизация
- **Axios** — HTTP‑клиент
- **Font Awesome** — иконки

### Инфраструктура

- **Docker** + **Docker Compose** — контейнеризация всего приложения
- **Nginx** — раздача статики фронтенда и проксирование API

## 📦 Возможности

- Регистрация и вход с сессионной аутентификацией (cookie)
- Просмотр товаров по категориям с фильтрацией (цена, сокет, тип памяти, бренд, рейтинг)
- Поиск товаров по названию
- Страница товара с характеристиками и описанием
- Корзина (сохраняется в БД для каждого пользователя)
- Оформление заказа с выбором доставки (самовывоз с картой / платная доставка) и оплаты (онлайн / при получении / кредит)
- Валидация банковской карты (Visa, Mastercard, Мир) и паспортных данных для кредита
- Страница профиля: личные данные, история заказов, сохранённые сборки ПК, избранное, обратная связь
- Конструктор сборки ПК с проверкой совместимости компонентов (сокет, тип памяти, мощность БП, форм‑фактор)
- Сохранение и загрузка сборок
- Хлебные крошки для удобной навигации
- Кэширование данных на клиенте (localStorage)
- Автоматическое продление сессии (пинг каждые 14 минут)

## 🛠️ Установка и запуск

### 1. Клонирование репозитория

```bash
https://github.com/KALIZEI611/HappyPC_Vue_Go.git
cd HappyPC_Vue_Go
```

### 2. Настройка переменных окружения (опционально)

По умолчанию используются значения из `backend/internal/config/config.go`. При необходимости переопределите их через файл `.env` в корне проекта или через переменные окружения в `docker-compose.yml`.

Основные переменные:

| Переменная  | Значение по умолчанию | Описание        |
| ----------- | --------------------- | --------------- |
| DB_HOST     | postgres              | хост PostgreSQL |
| DB_PORT     | 5432                  | порт            |
| DB_USER     | postgres              | пользователь БД |
| DB_PASSWORD | postgres              | пароль          |
| DB_NAME     | happypc               | имя БД          |
| SERVER_PORT | 8080                  | порт бэкенда    |

Для продакшена обязательно задайте свои пароли и укажите домен фронтенда в CORS.

### 3. Запуск через Docker Compose (рекомендуется)

```bash
docker-compose up --build
```

После успешной сборки и запуска:

- Фронтенд доступен по адресу **http://localhost**
- Бэкенд API — **http://localhost:8080**
- PostgreSQL — **localhost:5432**

Остановка: `Ctrl+C` или `docker-compose down`.

Для полной очистки (включая тома БД): `docker-compose down -v`.

### 4. Запуск в режиме разработки (без Docker)

#### Бэкенд

```bash
cd backend
go mod download
go run cmd/server/main.go
```

Сервер запустится на порту 8080. Убедитесь, что PostgreSQL запущен локально и параметры подключения (host=localhost) заданы в переменных окружения или в `config.go`.

#### Фронтенд

```bash
cd frontend
npm install
npm run dev
```

Приложение будет доступно на http://localhost:5173. В этом режиме работает прокси Vite (настроен в `vite.config.js`), поэтому все API‑запросы будут автоматически направляться на `http://localhost:8080`.

### 5. Создание администратора (опционально)

По умолчанию все пользователи имеют одинаковые права. Для ручного добавления администратора выполните SQL‑запрос в БД:

```sql
UPDATE users SET is_admin = true WHERE email = 'admin@example.com';
```

(Поле `is_admin` нужно добавить в модель `User`, если требуется разделение прав.)

## ⚙️ Возможные проблемы и их решение

### 1. Ошибка при сборке бэкенда: `cannot refer to unexported field db`

Убедитесь, что в `handlers/build.go` используется переданный `db` или метод репозитория, а не прямое обращение к неэкспортированному полю. В актуальной версии кода эта проблема исправлена.

### 2. CORS при работе из браузера

Если фронтенд и бэкенд запущены на разных портах, проверьте настройки CORS в `main.go`. Для разработки разрешены origins: `http://localhost`, `http://localhost:5173`. Для продакшена укажите конкретный домен.

### 3. Ошибка подключения к PostgreSQL в Docker

Убедитесь, что контейнер `happypc_db` запущен и здоров (смотрите `docker ps`). При первом запуске может потребоваться несколько секунд для инициализации. Если возникает ошибка `dial tcp 127.0.0.1:5432: connect: connection refused`, подождите 10–15 секунд и перезапустите `docker-compose up`.

### 4. Пустые страницы или 404 при обновлении SPA

Nginx уже настроен на fallback к `index.html` (в `nginx.conf`). Если вы используете другой сервер, убедитесь, что он перенаправляет все не‑существующие пути на `index.html`.

### 5. Кэширование и обновление данных

На клиенте используется `localStorage` для кэширования категорий, товаров и сборок. Если после изменения данных в БД страницы не обновляются, очистите localStorage (F12 → Application → Local Storage → удалите ключи `homeCategories`, `allProducts` и т.д.) или перезапустите контейнеры с флагом `-v`.
