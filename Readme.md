
# 🖥️ HappyPC — Интернет-магазин компьютерных комплектующих

**HappyPC** — это полноценный веб-сайт для подбора и покупки компьютерных комплектующих. Позволяет собирать ПК, управлять корзиной, оформлять заказы, сохранять избранное и многое другое.

---

## ✨ Что умеет приложение

- **Регистрация и вход** (сессии через cookie)
- **Каталог товаров** с категориями, фильтрами (цена, сокет, тип памяти, бренд, рейтинг) и поиском
- **Карточка товара** с характеристиками и описанием
- **Корзина** с возможностью изменения количества товаров
- **Оформление заказа** с выбором:
  - Самовывоз (с картой на карте) или платная доставка
  - Оплата онлайн, при получении или в кредит (с валидацией карты/паспорта)
- **Личный кабинет**:
  - Данные профиля
  - История заказов
  - Сохранённые сборки ПК
  - Избранное
  - Форма обратной связи
- **Конструктор ПК** с проверкой совместимости (сокет, тип памяти, мощность БП, форм-фактор)
- **Хлебные крошки** для удобной навигации
- **Кэширование данных** на клиенте (localStorage) для ускорения работы

## 🚀 Технологический стек

| Компонент       | Технологии                                                                                             |
|----------------|--------------------------------------------------------------------------------------------------------|
| **Бэкенд**     | Go (1.21+), Chi (роутер), GORM (ORM), PostgreSQL, bcrypt, сессии на основе cookie                      |
| **Фронтенд**   | Vue 3 (Composition API), Vite, Vue Router, Axios, Font Awesome                                         |
| **Инфраструктура** | Docker + Docker Compose (контейнеризация), Nginx (раздача статики фронтенда и проксирование API)    |

## 📦 Запуск проекта

### **1. Клонируйте репозиторий**

```bash
git clone https://github.com/KALIZEI611/HappyPC_Vue_Go.git
cd HappyPC_Vue_Go
```

### **2. Запуск через Docker (рекомендуется)**

Соберите и запустите все три сервиса (база данных, бэкенд, фронтенд):

```bash
docker-compose up --build
```

После успешного запуска:

- **Фронтенд** → [http://localhost](http://localhost)
- **Бэкенд API** → [http://localhost:8080](http://localhost:8080)
- **База данных** → `localhost:5432`

**Остановка:**

```bash
docker-compose down
```

Чтобы полностью удалить тома с данными БД:

```bash
docker-compose down -v
```

### **3. Ручной запуск (для разработки)**

#### 🔧 Бэкенд

```bash
cd backend
go mod download
go run cmd/server/main.go
```

Сервер запустится на порту `8080`.  
Требуется работающий PostgreSQL. Настройки подключения берутся из переменных окружения (по умолчанию: `localhost`, пользователь `postgres`, БД `happypc`).

#### 🎨 Фронтенд

```bash
cd frontend
npm install
npm run dev
```

Приложение будет доступно по адресу [http://localhost:5173](http://localhost:5173).  
Vite настроен на проксирование запросов к API на `http://localhost:8080`.

## ⚙️ Настройка переменных окружения (опционально)

По умолчанию используются следующие значения. При необходимости переопределите их в файле `.env` или через переменные окружения.

| Переменная       | Значение по умолчанию | Описание                |
|-----------------|-----------------------|-------------------------|
| `DB_HOST`       | `postgres` (в Docker) | Хост PostgreSQL         |
| `DB_PORT`       | `5432`                | Порт                    |
| `DB_USER`       | `postgres`            | Пользователь БД         |
| `DB_PASSWORD`   | `postgres`            | Пароль                  |
| `DB_NAME`       | `happypc`             | Имя базы данных         |
| `SERVER_PORT`   | `8080`                | Порт бэкенда            |
| `GIN_MODE`      | `release`             | Режим Gin (для Go)      |

> Для продакшена обязательно задайте сложные пароли и укажите свой домен фронтенда в `AllowedOrigins` (файл `main.go`).

## 🧪 Возможные проблемы и их решения

### 1. Ошибка подключения к БД в Docker

Убедитесь, что контейнер `happypc_db` запустился (команда `docker ps`). Иногда требуется 10–15 секунд на инициализацию. Если возникает ошибка `connection refused` — подождите и перезапустите `docker-compose up`.

### 2. Ошибка сборки бэкенда `cannot refer to unexported field db`

В актуальной версии кода эта проблема исправлена. Убедитесь, что вы используете последний коммит.

### 3. CORS при разработке

Для локальной разработки разрешены источники: `http://localhost` и `http://localhost:5173`. Если вы запускаете фронтенд на другом порту или домене — добавьте его в `AllowedOrigins` в `main.go`.

### 4. Пустые страницы или 404 при обновлении SPA

Nginx уже настроен на fallback к `index.html`. При ручном развёртывании убедитесь, что ваш веб-сервер аналогично перенаправляет все запросы на `index.html`.

### 5. Не обновляются данные после изменений в БД

На клиенте используется кэш в `localStorage`. Чтобы принудительно обновить данные, откройте DevTools (F12) → **Application** → **Local Storage** и удалите ключи `homeCategories`, `allProducts` и т.д.
